# ADR-003: Гибридная модель характеристик инструментов

**Статус:** Принято

**Дата:** 27.05.2026

## Контекст

Фиксированные колонки (`TickSize`, `LotMultiplier`) покрывают 80% потребностей,
но не позволяют добавлять произвольные атрибуты (ГО, экспирация, сектор, Greeks).

## Решение

Гибрид:

1. **Жёсткое ядро:** `InstrumentSpec` с полями для P&L-расчётов
2. **Гибкий шлейф:** EAV-таблицы `CharacteristicDefinitions` + `InstrumentCharacteristics`

Ядро — для скорости и строгой типизации.
Шлейф — для кастомизации, отчётов, пользовательских атрибутов.

В MVP EAV-таблицы создаются, но UI для редактирования — в v1.1.
